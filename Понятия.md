# repo
Это то, что работает с данными и проводит над ними круд

# gateway
Это порт, что взаимодействует с внешним миром (не в смысле обязательно с внешним апи или ресурсом, а в смысле за рамками бизнес логики и сущностей) и отдает данные в usecase
repo, например, частный случай gateway

# Value Object
Это структурный объект домена, который несет определенную информацию константами и помечается как dataclass(frozen=True). Например, ты хочешь определить структуру тестов для сбора метрик модели машинного обучения и хочешь определить вид данных как вопрос-ответ. Это будут объекты типа
```python
@dataclass(frozen=True)
class MetricTest:
	question: str
	answer: str
```
И где-то хранишь много таких экземпляров и имеешь например маппер из json в список этих value object
